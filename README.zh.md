# F1 队友中位排位差

[English](./README.md) | [日本語](./README.ja.md) | 简体中文

[**启动应用**](https://hsu9ar.github.io/f1-teammate-quali-gap/)

这是一个用于比较 F1 队友排位速度的交互式网页工具。它使用**中位排位差**来降低撞车、黄旗、机械故障、湿地或明显受干扰圈速对结论的影响。

## 这个工具能做什么

- 从免费的 Jolpica-F1 API 加载 F1 排位结果（自动回退至 CORS 代理 → localStorage 缓存 → 捆绑离线快照）。
- 选择赛季、车队和两位队友进行比较。
- 计算每站百分比差距、赛季中位排位差、赛季平均差、队内对决比分和有效样本数。
- 用柱状图展示每站差距,并提供针对极端值的聚焦视图。
- 点击图表柱子或表格行即可剔除不具代表性的站点。
- 支持英文、日文、简体中文界面。

## 如何使用

1. 点击上方启动链接打开应用。
2. 选择赛季、车队、车手 A 和车手 B。
3. 查看上方卡片中的中位排位差、平均差、对决比分和有效样本数。
4. 通过柱状图观察每站差距变化。
5. 遇到湿地、事故、机械问题或明显不代表真实速度的样本时,点击对应柱子或表格行将其剔除。
6. 使用 **聚焦视图** 在可读性更好的极值裁剪图表和完整尺度图表之间切换。

## 如何理解指标

差距以车手 A 相对车手 B 的圈速百分比表示:

```text
gap% = (t_A - t_B) / t_B * 100
```

- 正值:车手 A 慢于车手 B。
- 负值:车手 A 快于车手 B。
- 中位排位差:把所有有效站点差距排序后得到的典型赛季排位差距。
- 平均差:可以作为参考,但更容易被极端样本带偏。

## 计算方法

对每一站排位:

1. 找到两位车手都进入的最高共同阶段:优先 Q3,其次 Q2,最后 Q1。
2. 取两人在该共同阶段的最快圈。
3. 将圈速差换算为百分比。
4. 对所有未剔除站点的百分比差距取赛季中位排位差。

这样可以避免把 Q3 圈和 Q1 圈直接对比,也能消除不同赛道长度对绝对秒差的影响。

## 数据源

本项目使用免费的 [Jolpica-F1 API](https://github.com/jolpica/jolpica-f1)。它是 Ergast API 的社区后继项目,不需要 API key。

当实时 API 不可用时,应用会依次回退至 CORS 代理、上次成功加载的 localStorage 缓存,以及由每日 GitHub Actions 工作流生成的捆绑离线快照（`data/quali-<year>.json`）。

## 局限性

- 数据源不会标注干地和湿地。
- 删除圈、交通、黄旗、机械问题等情况仍需要手动判断和剔除。
- 该指标只反映排位和单圈速度,不衡量正赛速度、轮胎管理或策略执行。
- 若实时 API、CORS 代理、缓存和离线快照均不可用，则无法加载数据。
