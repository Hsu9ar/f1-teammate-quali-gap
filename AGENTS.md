# AGENTS.md

## Project Goal

Build a static, browser-based F1 data analysis tool that compares two teammates' qualifying pace across a season using median qualifying gap. The project is intended to be easy to open from GitHub Pages and understandable from the README alone.

## Current Progress

The core app is implemented as a single HTML/CSS/JavaScript file. It fetches live qualifying data from the free Jolpica-F1 API, calculates teammate gaps, visualizes per-race variation, and supports English, Japanese, and Simplified Chinese UI.

## Key Files

- `index.html`: GitHub Pages entry point. Redirects to the main app.
- `f1-teammate-quali-gap.html`: Main static app, including UI, i18n, data fetching, calculations, chart, and table.
- `tools/build-snapshots.mjs`: Local Node script that fetches season qualifying data and writes `data/quali-<year>.json` offline snapshots. Run on a machine that can reach the API; commit the resulting files.
- `data/quali-<year>.json`: Bundled offline snapshots (processed `{year, rounds}` shape). Last-resort fallback used only when the live API and proxies are unreachable. Pre-stored in the repo so no live visit is needed to populate them.
- `.github/workflows/update-snapshots.yml`: GitHub Actions workflow that runs the generator on GitHub's servers (weekly + manual) and commits refreshed `data/` snapshots. This keeps the offline data current in the cloud without any user visit.

Scope note: this repo only backs up **qualifying** data (2006 → current season), since that is all the app uses. Past seasons are fetched once and skipped on later runs; only the current season refreshes weekly. All other Jolpica datasets (results, race laps, pitstops, standings, sprint, schedules, drivers, constructors, circuits) are intentionally NOT stored here — they belong in a separate, dedicated F1 data-backup repo so this Pages app stays small. Jolpica has no practice-session data; practice long-run analysis requires a separate FastF1 (Python) pipeline.
- `README.md`: English project documentation.
- `README.ja.md`: Japanese project documentation.
- `README.zh.md`: Simplified Chinese project documentation.
- `.gitignore`: Ignores local cache and OS files.

## Important Conventions

- Keep the app static. Do not introduce a build step unless explicitly requested.
- Keep the primary app in `f1-teammate-quali-gap.html`.
- Use `index.html` only as the GitHub Pages entry point.
- The live Jolpica-F1 API is the primary data source. A resilient fallback chain was explicitly requested and added: live API (retry/backoff) → public CORS proxies → `localStorage` cache of prior successful loads → bundled `data/quali-<year>.json` snapshot. Keep this chain; do not strip the offline fallbacks. The status line shows when cached/snapshot data is being served.
- Snapshots are generated by `tools/build-snapshots.mjs` (not the old `build_data.py`). Do not auto-load snapshots ahead of the live API — they are last-resort only.
- Chinese documentation and UI should use Simplified Chinese.
- In Chinese, the metric should be called `中位排位差`.
- Avoid adding personal instructions, portfolio notes, or owner-specific setup text to public README files.
